## Общая концепция

**RadarTcell** — это backend для веб-платформы визуализации 100+ перспективных технологий на интерактивном радаре.

### Ключевые фичи:

- **Радарная визуализация**: каждая технология отображается точкой на круговой диаграмме
  - **Угол (angle)** = тренд (AI, IoT, Biotech...)
  - **Радиус (radius)** = TRL (Technology Readiness Level, 1-9)
  - **Размер пузыря** = custom_metric_1 или custom_metric_2
  - **Высота бара** = custom_metric_3 или custom_metric_4

- **Фильтрация и поиск**: по тренду, SDG (цели устойчивого развития), тегам, организациям, TRL
- **Highlight**: подсветка связанных технологий (`highlight=tag:ml&highlight=trend:ai`)
- **Сортировка**: по имени, TRL, любой orderable-метрике
- **Админ-панель**: CRUD для всех сущностей через JWT-авторизацию
- **Загрузка файлов**: изображения технологий/логотипы организаций в MinIO/S3
- **Мультиязычность**: поддержка локалей через i18n-таблицы

### Слои:

1. **Domain** (`internal/domain/`) — модели данных, бизнес-правила, валидация
2. **Ports** (`internal/ports/`) — интерфейсы (Repository, Service, Cache, Storage)
3. **Service** (`internal/service/`) — бизнес-логика (технологии, каталог, auth)
4. **Repository** (`internal/repository/postgres/`) — работа с БД
5. **HTTP API** (`internal/httpapi/`) — обработчики, middleware, роутер
6. **App** (`internal/app/`) — сборка зависимостей (DI)
7. **Infrastructure** — Redis (cache), MinIO (storage), Prometheus (metrics)

### Пример: `GET /api/technologies?search=ai&trl_min=5&sort_by=name`

```
1. HTTP-запрос приходит на :8080
   ↓
2. chi.Router → middleware: RequestID, Logger, CORS, CSRF
   ↓
3. Роут совпадает: GET /api/technologies → tech.List
   ↓
4. TechnologyHandler.List (internal/httpapi/technology_handler.go)
   - Парсит query-параметры (search, trl_min, sort_by...)
   - Валидирует строго (page>=1, limit 1..200, trl_min 1..9)
   - Если ошибка → 400 Bad Request
   ↓
5. TechnologyService.List (internal/service/technology_service.go)
   - Проверяет кэш Redis: ключ = "techs:v1:page=1&search=ai&trl_min=5&sort_by=name"
   - Если в кэше → возвращает
   - Если нет:
     ↓
6. TechnologyRepository.ListTechnologies (internal/repository/postgres/technology_repo.go)
   - Формирует SQL:
     SELECT tech.*, tr.name FROM technologies tech
     JOIN trends tr ON tr.id = tech.trend_id
     WHERE tech.name ILIKE '%ai%' AND tech.readiness_level >= 5
     ORDER BY tech.name ASC
     LIMIT 20 OFFSET 0
   - Выполняет запрос к PostgreSQL
   - Возвращает []domain.Technology + total count
   ↓
7. TechnologyService вычисляет:
   - Координаты радара (angle, radius)
   - Нормализованные метрики (custom_metric_1_norm, ...)

   ↓
8. Handler отправляет JSON:
```

## Как работает радарная логика

### 1. Расчёт координат (service/radar_coords.go)

#### **Радиус (distance от центра)**

TRL (1-9) → нормализуется в [0, 1]:

```go
func computeRadius(trl int) float64 {
    if trl < 1 { return 0 }
    if trl > 9 { return 1 }
    return float64(trl - 1) / 8.0
}

// Примеры:
// TRL 1 (Initial Research) → radius = 0.0 (центр)
// TRL 5 (Prototype) → radius = 0.5 (середина)
// TRL 9 (Production) → radius = 1.0 (край)
```

#### **Угол (позиция на круге)**

Зависит от **тренда** (AI, IoT, Biotech...):

```go
// 1. Получаем упорядоченный список всех трендов
trendIDs := []string{"trend-ai", "trend-iot", "trend-bio", ...} // 10 трендов

// 2. Вычисляем ширину сегмента
segmentWidth := 360.0 / len(trendIDs) // = 36° для 10 трендов

// 3. Для каждого тренда — базовый угол
trendPos := map[string]int{
    "trend-ai": 0,   // 0°
    "trend-iot": 1,  // 36°
    "trend-bio": 2,  // 72°
}

// 4. Внутри тренда — рандомизируем (по slug технологии)
func computeAngle(trendID, techSlug string) float64 {
    baseAngle := trendPos[trendID] * segmentWidth
    offset := hashSlug(techSlug) % segmentWidth // детерминированное смещение
    return baseAngle + offset
}

// Пример:
// Технология "chatgpt" в тренде AI (0°):
//   angle = 0 + hash("chatgpt") % 36 = 23.4°
```

### 2. Нормализация метрик (для размера пузыря/бара)

```go
// Находим min/max среди всех технологий
values := []float64{tech1.CustomMetric1, tech2.CustomMetric1, ...}
min := 0.5
max := 10.0

// Нормализуем каждую в [0, 1]
func normalize(v, min, max float64) float64 {
    if max == min { return 1.0 } // все одинаковые
    return (v - min) / (max - min)
}

// Пример:
// custom_metric_1 = 7.0 → norm = (7-0.5)/(10-0.5) = 0.68
```

### 3. Highlight-фильтрация

Клиент передаёт `?highlight=tag:ml&highlight=trend:ai`:

```go
// 1. Парсим highlight-параметры
highlights := []string{"tag:ml", "trend:ai"}

// 2. Для "tag:ml": находим ID тега "ml"

// 4. То же для "trend:ai"

// 5. Объединяем и передаём в ListTechnologies как фильтр OnlyIDs

// SQL превращается в:
// WHERE tech.id::text = ANY($5::text[])
```

## Безопасность и авторизация

### 1. JWT-авторизация (service/auth_service.go)

```go
type AuthService struct {
    adminUser     string
    adminPassword string
    secret        []byte
    ttl           time.Duration
}

// Логин: проверяет credentials, выдаёт JWT
func (s *AuthService) Login(ctx context.Context, username, password string) (string, bool, error) {
    if username != s.adminUser || password != s.adminPassword {
        return "", false, nil // bad credentials
    }
    
    // Генерируем JWT
    claims := jwt.RegisteredClaims{
        Subject: username,
        IssuedAt: jwt.NewNumericDate(time.Now()),
        ExpiresAt: jwt.NewNumericDate(time.Now().Add(s.ttl)), // 8 часов
    }
    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    signed, _ := token.SignedString(s.secret)
    
    return signed, true, nil
}

// Verify: проверяет JWT
func (s *AuthService) Verify(ctx context.Context, token string) (string, bool, error) {
    parsed, err := jwt.ParseWithClaims(token, &jwt.RegisteredClaims{}, func(t *jwt.Token) (any, error) {
        return s.secret, nil
    })
    
    if err || !parsed.Valid {
        return "", false, nil
    }
    
    claims := parsed.Claims.(*jwt.RegisteredClaims)
    return claims.Subject, true, nil
}
```

### 2. Middleware AuthRequired (httpapi/mw_auth.go)

```go
func AuthRequired(auth ports.AuthService) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            // 1. Читаем заголовок Authorization
            h := r.Header.Get("Authorization")
            if !strings.HasPrefix(h, "Bearer ") {
                writeError(w, 401, "missing bearer token")
                return
            }
            
            // 2. Извлекаем токен
            token := strings.TrimSpace(strings.TrimPrefix(h, "Bearer "))
            
            // 3. Проверяем
            sub, ok, err := auth.Verify(r.Context(), token)
            if err != nil {
                writeInternalError(w) // 500
                return
            }
            if !ok {
                writeError(w, 401, "invalid token")
                return
            }
            
            // 4. Сохраняем subject в контексте
            ctx := context.WithValue(r.Context(), ctxAdminSubject, sub)
            next.ServeHTTP(w, r.WithContext(ctx))
        })
    }
}
```

### 3. CORS/CSRF защита (httpapi/mw_cors_csrf.go)

#### CORS: разрешаем запросы с фронтенда

```go
func CORS(cfg CORSConfig) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            origin := r.Header.Get("Origin")
            
            // Проверяем, что origin в whitelist
            if isOriginAllowed(origin, cfg.AllowedOrigins) {
                w.Header().Set("Access-Control-Allow-Origin", origin)
                w.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE")
                w.Header().Set("Access-Control-Allow-Headers", "Authorization, Content-Type")
                w.Header().Set("Access-Control-Allow-Credentials", "true")
            }
            
            // Preflight (OPTIONS) → 204
            if r.Method == "OPTIONS" {
                w.WriteHeader(204)
                return
            }
            
            next.ServeHTTP(w, r)
        })
    }
}
```

#### CSRF: проверяем Origin/Referer для state-changing запросов

```go
func CSRF(cfg CSRFConfig) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            // GET/HEAD → пропускаем
            if !isStateChanging(r.Method) { // POST/PUT/DELETE
                next.ServeHTTP(w, r)
                return
            }
            
            origin := r.Header.Get("Origin")
            if origin != "" {
                if !isOriginAllowed(origin, cfg.TrustedOrigins) {
                    writeError(w, 403, "csrf: origin not allowed")
                    return
                }
            } else {
                // Проверяем Referer
                referer := r.Header.Get("Referer")
                if !isOriginAllowed(extractOrigin(referer), cfg.TrustedOrigins) {
                    writeError(w, 403, "csrf: referer not allowed")
                    return
                }
            }
            
            next.ServeHTTP(w, r)
        })
    }
}
```

## Загрузка файлов в MinIO

### 1. MinIO Storage (internal/storage/minio_storage.go)

```go
type MinioStorage struct {
    client     *minio.Client
    bucketName string
    publicURL  string
}

func NewMinioStorage(endpoint, accessKey, secretKey, bucket, publicURL string, useSSL bool) (*MinioStorage, error) {
    client, err := minio.New(endpoint, &minio.Options{
        Creds: credentials.NewStaticV4(accessKey, secretKey, ""),
        Secure: useSSL,
    })
    return &MinioStorage{client: client, bucketName: bucket, publicURL: publicURL}, nil
}

// Создаёт bucket и устанавливает публичный доступ
func (s *MinioStorage) EnsureBucket(ctx context.Context) error {
    exists, _ := s.client.BucketExists(ctx, s.bucketName)
    if !exists {
        s.client.MakeBucket(ctx, s.bucketName, minio.MakeBucketOptions{})
        
        // Public read policy
        policy := `{
          "Version": "2012-10-17",
          "Statement": [{
            "Effect": "Allow",
            "Principal": {"AWS": ["*"]},
            "Action": ["s3:GetObject"],
            "Resource": ["arn:aws:s3:::` + s.bucketName + `/*"]
          }]
        }`
        s.client.SetBucketPolicy(ctx, s.bucketName, policy)
    }
    return nil
}

// Загружает файл, возвращает публичный URL
func (s *MinioStorage) Upload(ctx context.Context, filename, contentType string, size int64, data io.Reader) (string, error) {
    // Генерируем уникальное имя: uuid-timestamp.ext
    ext := filepath.Ext(filename)
    objectName := uuid.New().String() + "-" + strconv.FormatInt(time.Now().Unix(), 10) + ext
    
    // Загружаем в MinIO
    _, err := s.client.PutObject(ctx, s.bucketName, objectName, data, size, minio.PutObjectOptions{
        ContentType: contentType,
    })
    
    // Формируем публичный URL
    url := s.publicURL + "/" + s.bucketName + "/" + objectName
    return url, err
}
```

### 2. Upload Handler (httpapi/upload_handler.go)

```go
func (h *UploadHandler) Upload(w http.ResponseWriter, r *http.Request) {
    // 1. Парсим multipart/form-data (лимит 10MB)
    r.ParseMultipartForm(10 << 20)
    
    // 2. Читаем файл
    file, header, err := r.FormFile("file")
    defer file.Close()
    
    // 3. Валидация: только изображения
    contentType := header.Header.Get("Content-Type")
    if !strings.HasPrefix(contentType, "image/") {
        writeError(w, 400, "only images allowed")
        return
    }
    
    // 4. Загружаем в MinIO
    url, err := h.storage.Upload(r.Context(), header.Filename, contentType, header.Size, file)
    if err != nil {
        writeInternalError(w)
        return
    }
    
    // 5. Возвращаем URL
    writeJSON(w, 200, map[string]string{"url": url})
}
```

```go
// internal/httpapi/mw_logging.go
func StructuredLogger(logger *zap.Logger) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            start := time.Now()
            
            ww := middleware.NewWrapResponseWriter(w, r.ProtoMajor)
            next.ServeHTTP(ww, r)
            
            logger.Info("http request",
                zap.String("method", r.Method),
                zap.String("path", r.URL.Path),
                zap.Int("status", ww.Status()),
                zap.Duration("duration", time.Since(start)),
                zap.String("ip", r.RemoteAddr),
            )
        })
    }
}
```

## Резюме: что происходит при запуске

1. **Загружаем .env** → читаем DATABASE_URL, REDIS_ADDR, MINIO_*, JWT_SECRET
2. **Подключаемся к PostgreSQL** → проверяем Ping
3. **Инициализируем Redis** (если настроен) → кэш на 60-300 сек
4. **Инициализируем MinIO** (если настроен) → создаём bucket с публичным доступом
5. **Создаём сервисы** → CatalogService, TechnologyService, AuthService...
6. **Собираем роутер** → добавляем middleware (CORS, CSRF, Auth, Logger)
7. **Регистрируем эндпоинты**:
   - Публичные: `/api/technologies`, `/api/trends`, ...
   - Админские: `/api/admin/login`, `/api/admin/technologies`, `/api/admin/upload`
8. **Запускаем HTTP-сервер** на :8080
9. **При запросе**:
   - Middleware → Logger → CORS → CSRF → AuthRequired (если admin)
   - Handler → Service → Repository → PostgreSQL
   - Если кэш → Redis HIT
   - Если нет → вычисляем координаты → сохраняем в Redis
   - Возвращаем JSON
