## 🎯 КЛЮЧЕВЫЕ ПОНЯТИЯ

### 🔹 Domain (Домен)
Модели данных и бизнес-логика. Не зависит от HTTP, БД, фреймворков.

### 🔹 Port (Порт)
Интерфейс - контракт между слоями. Говорит "ЧТО нужно сделать", но не "КАК".

### 🔹 Adapter (Адаптер)
Реализация интерфейса. Например, PostgresRepository - это адаптер для работы с БД.

### 🔹 Service (Сервис)
Бизнес-логика приложения. Оркестрирует вызовы репозиториев, применяет правила.

### 🔹 Repository (Репозиторий)
Абстракция работы с данными. Скрывает детали SQL, предоставляет простые методы.

### 🔹 Handler (Обработчик)
Обрабатывает HTTP-запросы. Парсит параметры, вызывает сервисы, формирует ответ.

### 🔹 Middleware (Посредник)
Функция, которая выполняется до/после обработчика. Логирование, авторизация, и т.д.

### 🔹 Migration (Миграция)
Скрипт изменения структуры БД. Версионирует схему базы данных.

### 🔹 Dependency Injection (Внедрение зависимостей)
Передача зависимостей через конструкторы, а не создание внутри компонента.

### 🔹 Clean Architecture (Чистая архитектура)
Принцип разделения на слои с зависимостями, направленными внутрь.
