# 📚 ПОЛНОЕ ОБЪЯСНЕНИЕ ПРОЕКТА RadarTcell

## 🎯 ЧТО ЭТО ЗА ПРОЕКТ?

**RadarTcell** — это веб-приложение для управления каталогом технологий. Это как "магазин технологий", где можно:
- Просматривать список технологий 
- Фильтровать их по разным критериям (тренды, теги, организации, SDG)
- Смотреть детальную информацию о каждой технологии
- Администраторы могут добавлять/редактировать/удалять данные

**Технологии:**
- **Язык:** Go (Golang)
- **База данных:** PostgreSQL
- **API:** REST API
- **Документация:** Swagger/OpenAPI

#### 📄 `main.go` — ГЛАВНЫЙ ФАЙЛ

**Что он делает?**

1. **Загружает переменные окружения** (строка 19):
   ```go

   ```
   Ищет файл `.env` и загружает из него настройки.

2. **Читает настройки** (строки 21-34):
   - `DATABASE_URL` — адрес базы данных (ОБЯЗАТЕЛЬНО!)
   - `APP_PORT` — порт для сервера (по умолчанию 8080)
   - `ADMIN_USER`, `ADMIN_PASSWORD` — логин/пароль админа
   - `JWT_SECRET` — секретный ключ для JWT токенов

3. **Подключается к базе данных** (строки 42-53):
   ```go
   pool, err := pgxpool.New(ctx, dbURL)

   Создает пул соединений к PostgreSQL.

func CalculateRadarCoords(trl int, index int, maxIndex int) (float64, float64) {
    // Магия математики для расчета позиции на круговой диаграмме
    radius := float64(trl) / 9.0 * 100
    angle := float64(index) / float64(maxIndex) * 2 * math.Pi
    
    x := radius * math.Cos(angle)
    y := radius * math.Sin(angle)
    
    return x, y
}
```

#### 📄 `mw_auth.go` — MIDDLEWARE ДЛЯ АВТОРИЗАЦИИ

Проверяет JWT-токен:

```go
func AuthMiddleware(authService ports.AuthService) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            // Читаем заголовок Authorization
            authHeader := r.Header.Get("Authorization")
            if authHeader == "" {
                writeError(w, http.StatusUnauthorized, "missing token")
                return
            }
            
            // Проверяем формат "Bearer <token>"
            token := strings.TrimPrefix(authHeader, "Bearer ")
            
            // Валидируем токен
            if err := authService.ValidateToken(token); err != nil {
                writeError(w, http.StatusUnauthorized, "invalid token")
                return
            }
            
            // Если все ОК, передаем управление дальше
            next.ServeHTTP(w, r)
        })
    }
}
```

## 🔄 КАК ВСЕ РАБОТАЕТ ВМЕСТЕ?

### Пример: Получение списка технологий

1. **Клиент** отправляет GET-запрос:
   ```
   GET /api/technologies?search=AI&trend_id=trend-1&page=1
   ```

2. **Router** (`router.go`) перенаправляет на обработчик:
   ```go
   api.Get("/technologies", tech.List)
   ```

3. **Handler** (`technology_handler.go`) парсит параметры:
   ```go
   params := domain.TechnologyListParams{
       Search:  r.URL.Query().Get("search"),
       TrendID: r.URL.Query().Get("trend_id"),
       Page:    1,
   }
   ```

4. **Service** (`technology_service.go`) добавляет валидацию:
   ```go
   if params.Page < 1 { params.Page = 1 }
   items, total, err := s.repo.FindAll(ctx, params)
   ```

5. **Repository** (`technology_repo.go`) делает SQL-запрос:
   ```go
   query := `
       SELECT id, slug, name, ...
       FROM technologies
       WHERE name ILIKE $1 AND trend_id = $2
       ORDER BY list_index
       LIMIT $3 OFFSET $4
   `
   rows, err := r.db.Query(ctx, query, "%"+params.Search+"%", params.TrendID, limit, offset)
   ```

6. **Handler** возвращает JSON:
   ```go
   writeJSON(w, http.StatusOK, map[string]any{
       "data": items,
       "total": total,
   })
   ```

## 🎨 АРХИТЕКТУРНЫЕ ПАТТЕРНЫ

### 1. **Clean Architecture (Чистая архитектура)**

Слои:
- **Domain** (внутренний) — модели данных
- **Ports** — интерфейсы
- **Service** — бизнес-логика
- **Repository** — работа с БД
- **HTTP API** (внешний) — HTTP-обработчики

### 2. **Dependency Injection (Внедрение зависимостей)**

Все зависимости передаются через конструкторы:

```go
func NewTechnologyService(repo ports.TechnologyRepository) *TechnologyService {
    return &TechnologyService{repo: repo}
}
```

### 3. **Repository Pattern**

Скрывает детали работы с БД:

```go
type TechnologyRepository interface {
    FindAll(ctx, params) ([]Technology, error)
    FindByID(ctx, id) (*Technology, error)
}
```

### 4. **Middleware Pattern**

Цепочка обработчиков:

```go
r.Use(middleware.Logger)      // Логирование
r.Use(middleware.Recoverer)   // Обработка паники
r.Use(AuthMiddleware)         // Проверка авторизации
```

## 📊 ОСНОВНЫЕ СУЩНОСТИ

### 1. **Technology (Технология)**
- Главная сущность
- Привязана к тренду
- Имеет уровень готовности (TRL)
- Может иметь метрики

### 2. **Trend (Тренд)**
- Категория технологий
- Примеры: AI, IoT, Blockchain

### 3. **SDG (Цели устойчивого развития)**
- 17 целей ООН
- Технологии могут быть привязаны к нескольким SDG

### 4. **Tag (Тег)**
- Метка для технологий
- Примеры: "машинное обучение", "большие данные"

### 5. **Organization (Организация)**
- Компании/институты
- Связаны с технологиями
- Имеют метрики

### 6. **Metric (Метрика)**
- Показатели организаций
- Примеры: "количество патентов", "инвестиции"

### 7. **User Preferences (Предпочтения)**
- Какие технологии интересны пользователю

## 🐛 ОБРАБОТКА ОШИБОК

### Уровни ошибок:

1. **Domain errors** (бизнес-ошибки):
   ```go
   if tech == nil {
       return domain.ErrNotFound
   }
   ```

2. **Service errors** (логические ошибки):
   ```go
   if params.Page < 1 {
       return nil, 0, errors.New("invalid page")
   }
   ```

3. **HTTP errors** (ошибки API):
   ```go
   if err == domain.ErrNotFound {
       writeError(w, http.StatusNotFound, "technology not found")
       return
   }
   ```
