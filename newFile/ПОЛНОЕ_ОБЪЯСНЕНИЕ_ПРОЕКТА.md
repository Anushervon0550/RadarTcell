# 📚 ПОЛНОЕ ОБЪЯСНЕНИЕ ПРОЕКТА RadarTcell

## ✅ КОРОТКО (1–2 минуты)

RadarTcell — это REST API для каталога технологий (тренды, SDG, теги, организации, метрики).
Сервер сам считает координаты радара, делает фильтрацию/сортировку и выдает готовые данные для фронтенда.

Важно:
- Админские эндпоинты: `/api/admin/*`.
- Локализация: `?locale=ru`.
- Системные: `/healthz`, `/readyz`, `/metrics`, `/openapi.yaml`, `/swagger/`.

---

## ✅ БЫСТРЫЙ СТАРТ

```powershell
cd C:\PROJECT\RadarTcell\deploy
docker compose up -d

cd C:\PROJECT\RadarTcell
migrate -path "./migrations" -database $env:DATABASE_URL up

go run .\cmd\api\main.go
```

Swagger: `http://localhost:8080/swagger/`

---

## 🧭 СТРУКТУРА ПРОЕКТА

- `cmd/api` — запуск приложения.
- `internal/domain` — доменные модели.
- `internal/ports` — интерфейсы.
- `internal/service` — бизнес‑логика.
- `internal/repository/postgres` — SQL и работа с БД.
- `internal/httpapi` — HTTP handlers и router.
- `migrations` — миграции БД.
- `docs` — OpenAPI/Swagger.
- `deploy` — docker-compose + мониторинг.

---

## 🔄 КАК ПРОХОДИТ ЗАПРОС

```
HTTP → Router → Handler → Service → Repository → PostgreSQL
                                      ↑
                                  (данные)
```

Service слой добавляет бизнес‑логику:
- расчет угла/радиуса на радаре,
- нормализацию метрик,
- фильтры/сортировку.

---

## 📦 ОСНОВНЫЕ СУЩНОСТИ

- **Technology**: name, TRL, метрики, связи.
- **Trend**: категория технологий.
- **SDG**: цели устойчивого развития.
- **Tag**: доменные метки.
- **Organization**: компании/институты.
- **MetricDefinition**: справочник метрик.
- **Preferences**: настройки пользователя.

---

## 📡 ПУБЛИЧНЫЕ API (коротко)

- `GET /api/technologies` — список с фильтрами/сортировкой.
- `GET /api/technologies/{slug}` — карточка технологии.
- `GET /api/trends`, `/api/sdgs`, `/api/tags`, `/api/organizations`, `/api/metrics`.
- `GET /api/*/{slug|code}/technologies` — связи.
- `POST /api/preferences`, `GET /api/preferences/{user_id}`.

Локализация: `?locale=ru`.

---

## 🔐 АДМИНСКИЕ API

- `POST /api/admin/login` → JWT.
- CRUD по всем сущностям.
- i18n‑переводы:

```
PUT /api/admin/i18n/trends/{slug}?locale=ru
PUT /api/admin/i18n/technologies/{slug}?locale=ru
PUT /api/admin/i18n/metrics/{id}?locale=ru
```

---

## 🧠 БИЗНЕС‑ЛОГИКА

- Радиус: TRL 1..9 → 0..1.
- Угол: сегмент тренда + стабильный hash от slug.
- Нормализация метрик: 0..1 для bubble/bar.
- `highlight` позволяет выделить связанные элементы.

---

## 🌍 ЛОКАЛИЗАЦИЯ

Публичное чтение: `?locale=ru`.
