## 📝 ЗАДАЧА 1: Добавить новое поле (пример — `author`)

### 1) Миграция

```sql
ALTER TABLE technologies ADD COLUMN author TEXT;
CREATE INDEX IF NOT EXISTS idx_technologies_author ON technologies(author);
```

### 2) Domain

`internal/domain/technology.go`

```go
Author *string
```

### 3) Repository

- Добавить поле в SELECT/INSERT/UPDATE.

### 4) Service + Handler

- Прокинуть поле без дополнительной логики.

### 5) OpenAPI

- Добавить `author` в схему технологии.

---

## 📝 ЗАДАЧА 2: Новый эндпоинт (пример — популярные технологии)

**Цель:** `GET /api/technologies/popular?limit=10`

1) Port: добавить `GetPopular` в `TechnologyService`.
2) Repository: SQL с `ORDER BY readiness_level DESC`.
3) Service: валидировать `limit`.
4) Handler: новый метод.
5) Router: маршрут до `/technologies/{slug}`.
6) OpenAPI: добавить path.

---

## 📝 ЗАДАЧА 3: Новый фильтр (пример — `author`)

**Цель:** `GET /api/technologies?author=IBM`

1) `Author` в `TechnologyListParams`.
2) `WHERE t.author ILIKE ...` в SQL.
3) Парсинг query в handler.
4) Параметр в OpenAPI.
